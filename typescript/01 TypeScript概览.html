# Types as Sets

In Java, itâ€™s meaningful to think of a one-to-one correspondence between runtime types and their compile-time declarations.

In TypeScript, itâ€™s better to think of a type as a set of values that share something in common. Because types are just sets, a particular value can belong to many sets at the same time.

  x: number;
  y: number;
}

  name: string;

function logPoint(point: Pointlike) {
  console.log("x = " + point.x + ", y = " + point.y);
}
